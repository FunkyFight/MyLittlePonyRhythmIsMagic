# Plan: corriger les dropdowns des fenêtres DevUI

## Constat
- `SeeSawEditorNoteOptionsPanel` et `SeaponyParadeNoteOptionsPanel` utilisent tous les deux `DevUiWindowRow.Dropdown`, donc le bug doit être corrigé au niveau générique `Elements/DevUI` plutôt que par un hack spécifique Seapony.
- `DevUiFloatingWindow` garde l'identité du dropdown ouvert dans `_openDropdownKey`, mais délègue aussi l'état ouvert à une seule instance mutable `DevUiDropdown` partagée par toutes les lignes.
- `UpdateDropdownRows` appelle actuellement `_dropdown.Update(...)` pour chaque dropdown visible. Comme `DevUiDropdown.Update` commence par `_isOpen = isOpenRow`, une ligne dropdown non ouverte peut désynchroniser `_dropdown.IsOpen` de `_openDropdownKey`.
- `MouseOverOpenDropdown` dépend ensuite de `_dropdown.IsOpen`, alors que le rendu de la liste dépend de `_openDropdownKey`. Résultat possible : la liste est dessinée comme ouverte, mais les clics/fermetures ne la considèrent plus comme ouverte.
- Le panneau Seapony modifie `AdditionnalData` différemment de See Saw. Ce n'est probablement pas la cause principale du clic bloqué, mais il faut l'aligner pour éviter une différence de comportement inutile.

## Correction prévue
1. Dans `Elements/DevUI/DevUiFloatingWindow.cs`, faire de `_openDropdownKey` la source de vérité de l'ouverture.
2. Modifier `MouseOverOpenDropdown` pour tester `_openDropdownKey != null` et les bounds réels du dropdown ouvert, sans dépendre de `_dropdown.IsOpen`.
3. Modifier `UpdateDropdownRows` pour ne pas appeler `_dropdown.Update` sur tous les dropdowns : seulement sur la ligne ouverte ou sur la ligne dont le champ dropdown vient d'être cliqué.
4. Quand un clic sélectionne une option, invoquer `row.Select`, fermer `_openDropdownKey`, et retourner `true` pour signaler une vraie modification.
5. Quand un clic est hors de la liste ouverte mais dans la fenêtre, fermer seulement le dropdown et consommer le clic si nécessaire, afin d'éviter que le clic tombe sur une autre ligne par accident.
6. Réinitialiser `_openDropdownKey` si la clé ouverte ne correspond plus à une ligne dropdown visible après reconstruction dynamique des rows.
7. Garder `DevUiDropdown` comme composant de rendu/interactions locales si possible, mais empêcher son état `_isOpen` d'être écrasé par des dropdowns non concernés. Si la correction devient plus claire, déplacer l'état d'ouverture entièrement dans `DevUiFloatingWindow` et rendre `DevUiDropdown` quasi stateless.
8. Dans `Elements/Editor/Notes/SeaponyParade/SeaponyParadeNoteOptionsPanel.cs`, remplacer la mutation directe répétée par un setter local similaire à See Saw : récupérer la note une fois, garantir un dictionnaire, écrire `action`, puis réassigner `AdditionnalData`.

## Validation prévue
- Lancer `dotnet build` après implémentation.
- Tester manuellement dans l'éditeur : ouvrir les options d'une note Seapony Parade existante, changer `VARIANT` vers `Star`, `Tap Tap`, puis `Swim`; le clic doit être accepté et la fenêtre ne doit plus rester bloquée.
- Tester le mode création Seapony Parade : changer la variante avant `CREATE`, puis créer la note.
- Tester la régression See Saw : changer `TARGET` vers `Outer`, `Inner`, `Opposite`, `Exit`; vérifier aussi le dropdown `OPPOSITE MODE` quand `Opposite` est sélectionné.
- Vérifier qu'un clic hors liste ferme proprement le dropdown sans fermer la fenêtre si le clic reste dans la fenêtre, et ferme la fenêtre si le clic est réellement à l'extérieur.
