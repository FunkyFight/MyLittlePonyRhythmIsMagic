# Plan: refonte complete editeur Heaven Studio-like

## Objectif

Remplacer l'editeur actuel, majoritairement clavier et monolithique, par un editeur clique/drag inspire Heaven Studio/Premiere Pro:

- top bar avec menus dropdowns: `File`, `Actions`, `Data`, `Tools`.
- zone haute avec viewport scene a scale 0.5, liste des rhythm games, palette de notes/options.
- zone basse avec timeline multi-tracks, marqueurs, graduations, clips de notes, drag/drop, move, resize.
- toutes les mutations passent par un command stack undo/redo.
- les anciennes variantes deviennent des types de clips selectionnables dans la palette.
- les clips continus generent des hits a intervalle regulier, sans que l'utilisateur manipule chaque hit.

## Constat codebase

- `Elements/Editor/BeatmapEditorElement.cs` est un monolithe de ~2200 lignes: input clavier, timeline, creation, options, playback, metadata et rendu sont melanges.
- `Elements/DevUI` ne fournit que primitives tres simples: label, dropdown, floating window, float input. Il manque boutons de topbar, menus, tabs, text input string, scroll lists, panels, context menus.
- `BeatmapEditorDocument` est deja beat-first et possede les bonnes bases: `ChartTempoMap`, placement, delete, move, save, migration v1, BPM changes.
- Le schema `Chart` est dans `..\..\MonogameLibs\RhythmConductor\RhythmConductor\Note\Chart.cs`. Il ne contient pas encore `Description`, warning flashing effects, tags, icon/rating data, tracks ou clips editeur.
- Le runtime ne lit que `Chart.Notes` et `Chart.Effects`; il faut donc compiler les nouveaux clips editeur en `ChartNote` avant playback/save.
- `Game1` instancie l'editeur sans lui passer le `SceneManager`; pour afficher un viewport scene embedded, l'editeur aura besoin d'un acces au rendu de la scene courante ou d'un render provider.
- Le modele actuel considere une beatmap comme un seul fichier XML. La refonte doit la traiter comme un package dossier, avec un XML interne et des assets image propres a la beatmap.

## Decisions techniques

- Garder `Chart.Notes` comme sortie runtime compatible avec `BeatmapPlayer`, `ChartPlayer`, See-Saw et Seapony Parade.
- Ajouter une couche persistante editor-only: `EditorTrack` + `EditorClip`, chargee par `BeatmapEditorDocument` et compilee vers `Chart.Notes`.
- Etendre le schema `Chart` dans `MonogameLibs/RhythmConductor` avec des champs optionnels. Les anciens XML restent chargeables car les champs absents auront leurs valeurs par defaut.
- Remplacer la notion de chart path seul par un package: `Beatmaps/<BeatmapName>/chart.xml` + dossier `assets/` pour icone/rating images et futurs assets.
- Stocker dans le XML uniquement des chemins relatifs au dossier beatmap pour les images, afin que le dossier reste deplacable/copiable.
- Les operations fichier (`New`, `Load`, `Save`) ne sont pas undoables. Toutes les mutations de chart/editor state le sont: create/delete/move/resize clip, edit clip data, tempo change, metadata, select music.
- Garder `Ctrl+S` comme raccourci demande. Les autres actions doivent etre accessibles par clic; des raccourcis standards optionnels peuvent appeler les memes commands sans redevenir le flux principal.

## Nouveau modele editeur

Ajouter dans `Elements/Editor`:

- `EditorTrack`: id, nom, index, couleur optionnelle, mute/lock eventuels plus tard.
- `EditorClip`: id, track index, start beat, length beats, rhythm game id, clip type id, clip category, data, input action.
- `EditorClipCategory`: `SingleHit`, `Continuous`, `NoHit`, `TempoChange` si on veut unifier les effets visuellement, sinon garder les BPM en marker lane.
- `EditorRhythmGameDefinition`: nom du rhythm game, liste de `EditorClipDefinition`, options panel.
- `EditorClipDefinition`: nom affiche, couleur, longueur par defaut, generation de runtime notes, options panel.
- `EditorClipCompiler`: transforme les clips en `ChartNote` en tenant compte de la `ChartTempoMap`.

Mapping initial:

- See Saw: `LongLong`, `ShortShort`, `LongShort`, `ShortLong` comme clips distincts, plus eventuellement `Exit` si necessaire pour l'etat See-Saw existant.
- Seapony Parade: `Swim`, `Roll`, `Tap Tap` comme clips distincts.
- `Swim`: clip continu, genere un hit tous les 2 beats pendant la longueur du clip.
- `Roll`: clip continu/groupe, conserve la logique actuelle de completion par multiple de 4 si elle reste souhaitee.
- `Tap Tap`: clip single ou pattern qui genere deux hits separes de 0.5 beat, comme la logique actuelle.
- `NoHit`: clip visuel/state-only pret pour les besoins futurs, ne genere pas de `ChartNote`.

## Beatmap package et persistence XML

Une beatmap devient un dossier, pas seulement un XML. Structure cible:

```text
Beatmaps/
  My Beatmap/
    chart.xml
    assets/
      icon.png
      ratings/
        try_again.png
        ok.png
        superb.png
```

Regles package:

- `BeatmapEditorDocument` conserve `PackagePath`, `ChartPath` et `AssetsPath`.
- `ChartPath` pointe vers `PackagePath/chart.xml` pour les nouveaux charts.
- `File > New Chart` cree le dossier package, `chart.xml`, les 10 tracks par defaut et le dossier `assets/`.
- `File > Load Chart` liste les dossiers beatmap contenant `chart.xml`; il peut aussi afficher les XML legacy pour migration.
- `File > Save Chart` compile les clips, synchronise `Chart.Notes`, puis serialize uniquement `chart.xml` dans le package.
- Les images selectionnees dans `Level Icon` et `Rating Screen` sont importees dans `assets/` via un `EditorAssetManager`.
- `EditorAssetManager` sanitize les noms, evite les collisions (`name_2.png`), accepte au minimum `.png`, `.jpg`, `.jpeg`, et retourne le chemin relatif a stocker dans le XML.
- Les chemins XML utilisent `/` comme separateur logique, meme sous Windows.
- `SongPath` reste externe au package dans cette iteration, car `Data > Select Music` pointe deja vers `Songs/`. L'import audio dans le dossier beatmap peut etre ajoute plus tard si voulu.
- Les assets orphelins ne sont pas supprimes automatiquement au debut; ajouter plus tard une action `Clean Unused Assets` evitera les pertes accidentelles.

Migration legacy:

- Charger un ancien `Beatmaps/foo.xml` reste possible.
- Au premier `Save`, migrer vers `Beatmaps/foo/chart.xml` et garder l'ancien XML intact ou en backup `foo.xml.legacy`.
- Les champs image absents restent vides; les anciennes beatmaps n'ont donc pas besoin d'assets.

Dans `Chart.cs`, ajouter des champs optionnels:

- `Description`.
- `FlashingEffectsWarning`.
- `Tags` sous forme de collection XML simple.
- `LevelIconPath`, relatif au package.
- `RatingHeader`.
- `RatingTryAgainMessage`, `RatingTryAgainImagePath` relatif au package.
- `RatingOkMessage`, `RatingOkImagePath` relatif au package.
- `RatingSuperbMessage`, `RatingSuperbImagePath` relatif au package.
- `EditorTracks`.
- `EditorClips`.

Migration:

- Si `EditorClips` existe, l'editeur les utilise comme source de verite et regenere `Chart.Notes`.
- Si `EditorClips` est absent, convertir les `Chart.Notes` existantes en clips legacy best-effort.
- La conversion legacy peut commencer simple: un `ChartNote` devient un clip `SingleHit` sur track 0, avec game/type deduits par `EditorNoteDefinitions.FromChartNote` et `AdditionnalData`.
- Pour les vieux Seapony `Tap Tap` ou series `Roll`, ne pas chercher a reconstruire parfaitement des clips continus au premier passage; la sauvegarde suivante pourra les normaliser sous forme de clips si detectables.

## Command stack

Ajouter `Elements/Editor/Commands`:

- `IEditorCommand` avec `Name`, `Execute(BeatmapEditorDocument document)`, `Undo(BeatmapEditorDocument document)`.
- `EditorCommandStack` avec deux stacks: undo et redo.
- Commandes initiales: `CreateClipCommand`, `DeleteClipCommand`, `MoveClipCommand`, `ResizeClipCommand`, `ChangeClipDataCommand`, `CreateTempoChangeCommand`, `MoveTempoChangeCommand`, `DeleteTempoChangeCommand`, `SetMetadataCommand`, `SetSongPathCommand`, `ImportBeatmapAssetCommand`.
- `CompositeEditorCommand` pour les operations multi-objets comme paste ou suppression multiple.
- Chaque commande marque le document dirty, regenere les notes runtime et preserve la position de playback si necessaire.

## DevUI cible

Refactorer/etendre `Elements/DevUI` sans introduire de framework externe:

- `DevUiInputState`: snapshot mouse/keyboard, left/right click pressed/released, wheel, pointer position.
- `DevUiButton` ou helper button stateless.
- `DevUiMenuBar` et `DevUiDropdownMenu`: bouton texte qui ouvre une liste de boutons.
- `DevUiPanel`: rectangle avec header optionnel, clipping logique et scroll.
- `DevUiTabs`: onglets horizontaux.
- `DevUiTextInput`: edition string, pas seulement float.
- `DevUiCheckbox`, `DevUiIconButton`, `DevUiList`.
- `DevUiContextMenu` pour clic droit sur clip.
- `DevUiDragDrop`: payload minimal pour drag depuis palette vers timeline.

Principe: les controles retournent des intentions (`clicked`, `selected`, `changed`) et ne mutent pas directement le document; l'editeur convertit ces intentions en commands.

## Layout cible

Dans `BeatmapEditorElement`, remplacer les positions hardcodees par un layout calcule chaque frame:

- `TopBar`: hauteur ~32 px.
- `UpperContainer`: de `TopBar.Bottom` jusqu'a environ 45% de la hauteur ecran.
- `ScenePreviewPanel`: premier tiers/gros panneau, rendu scene a scale 0.5, letterbox si necessaire.
- `RhythmGameListPanel`: liste textuelle `See Saw`, `Seapony Parade`.
- `PaletteOrOptionsPanel`: affiche les clips du game selectionne; devient options panel quand un clip est selectionne/clic droit.
- `TimelinePanel`: reste de l'ecran, avec header graduations, marker lane, labels tracks a gauche, lanes a droite.

Timeline:

- 10 tracks par defaut a la creation d'un chart.
- colonne gauche avec noms des tracks.
- zone droite avec pixels-per-beat, scroll horizontal, zoom par wheel/controls UI.
- marker lane pour BPM changes/markers.
- playhead draggable.
- clips rectangulaires, couleur par clip type, texte du clip, handles gauche/droite pour resize.

## Interactions

Selection tool par defaut:

- clic sur clip: selection.
- drag clip: move beat + track, snap a la grille.
- drag handle: resize length en beats.
- clic vide timeline: deplace playhead.
- drag playhead: scrub.
- clic droit clip: ouvre options/context menu et remplace la palette par le note options panel.
- drag depuis palette vers track: cree un clip via `CreateClipCommand`.
- `Copy`: copie le clip selectionne dans un clipboard editeur interne.
- `Paste`: colle a la position du playhead et track selectionnee via command.

Tempo Change tool:

- clic marker lane/timeline: cree un BPM change a beat snappe.
- drag marker BPM: deplace l'effet.
- clic droit marker: options BPM.

Topbar:

- `File > New Chart`: ouvre modal nom + options de base, cree un dossier package avec 10 tracks.
- `File > Load Chart`: ouvre picker des packages existants et des XML legacy migrables.
- `File > Save Chart (CTRL+S)`: compile clips, synchronise timings, importe assets pending, serialize `chart.xml`.
- `Actions > Undo/Redo/Copy/Paste`: appelle command stack/clipboard.
- `Data > Select Music`: ouvre picker `Songs/**/*.{wav,mp3,ogg}`.
- `Data > Level Properties`: ouvre ecran proprietes.
- `Tools > Selection/Tempo Change`: change l'outil actif.

## Level Properties

Ajouter un overlay/ecran interne ouvert par `Data > Level Properties`:

- Onglet `Properties`: Beatmap title, Beatmap author, Description, Flashing Effects Warning, Tags, Song Name, Song Artist.
- Onglet `Level Icon`: bouton import image, preview, chemin relatif dans `assets/`.
- Onglet `Rating Screen`: Header, sous-onglets `Try Again`, `OK`, `Superb`; chaque rating expose message, bouton import image et preview.
- Les champs texte/checkbox passent par `SetMetadataCommand` ou une commande dediee par champ.
- Les imports image passent par une commande `ImportBeatmapAssetCommand` qui copie dans le package et met a jour le chemin relatif.
- `Select Music` met a jour `Chart.SongPath`, `MusicName` si vide, et reconstruit le playback.

## Scene preview

- Modifier l'integration `Game1` pour passer un `SceneManager` ou `Func<Scene>` a `BeatmapEditorElement`.
- Ajouter un renderer offscreen dans l'editeur: `RenderTarget2D` de taille scene preview, draw de la scene courante avec camera existante, puis draw dans le panel a scale 0.5.
- Eviter que ce rendu embedded clear/presente le backbuffer comme `RenderViewport.RenderScene` actuel; utiliser `BeginCapture`/`SetRenderTarget` ou un helper dedie.
- A terme, quand l'editeur est actif, `Game1` peut ne plus dessiner la scene full-screen derriere l'UI, ou la garder comme fond selon rendu voulu. La preview embedded doit fonctionner dans les deux cas.

## Refactor BeatmapEditorDocument

- Ajouter une collection `EditorClips` et `EditorTracks` exposee en lecture controlee.
- Ajouter `PackagePath`, `ChartPath`, `AssetsPath`, et une distinction claire entre documents packages et documents XML legacy non encore migres.
- Ajouter `CompileClipsToRuntimeNotes()` appele apres chaque mutation clip et avant playback/save.
- Separer les operations legacy note/effect actuelles des operations clips. Garder les helpers tempo, BPM, save/load, migration.
- Ajouter `ImportAsset(sourcePath, targetSubfolder)` ou deleguer a `EditorAssetManager`, avec retour du chemin relatif stockable dans `Chart`.
- Remplacer `TryPlaceNotes` direct pour le nouveau flux par `TryCreateClip` + compilation.
- Garder les validations speciales See-Saw en les appliquant sur les notes generees par clip avant commit.
- Pour les BPM changes, garder `Chart.Effects` comme source de verite, mais les afficher dans la marker lane.

## Risques

- Reconstructing legacy continuous clips depuis des notes atomiques peut etre imparfait; acceptable si les anciens charts restent editables comme clips single-hit.
- Le rendu scene embedded peut necessiter un petit ajustement de `Game1`/`SceneManager`, donc la refonte depassera legerement `Elements/Editor` et `Elements/DevUI`.
- Le command pattern doit etre applique strictement aux mutations, sinon undo/redo deviendra incoherent rapidement.
- See-Saw a des contraintes contextuelles fortes; compiler les clips en notes puis reutiliser `SeeSawChartCompiler` comme validation evitera de dupliquer cette logique.
- Le passage XML seul -> dossier package impose de bien gerer la migration pour ne pas ecraser les anciennes beatmaps ni perdre les references d'images.
