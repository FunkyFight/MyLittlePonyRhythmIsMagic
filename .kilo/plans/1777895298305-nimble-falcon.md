# Plan: notes d'editeur multi-hits avec fenetre globale

## Contexte observe
- `EditorNoteDefinition` supporte deja `IEditorNotePlacementStrategy`, donc une definition peut produire plusieurs `EditorNotePlacement` a partir d'une seule note source.
- `BeatmapEditorElement.CreatePendingNote()` utilise cette strategie pour la creation simple, mais l'intervalle (`I`) contourne la strategie via `CreateIntervalPlacements()` hardcode.
- `BeatmapEditorDocument.TryPlaceNotes()` valide chaque note generee comme une note independante, avec des timings calcules depuis `Chart.Notes` uniquement. Les notes generees dans la meme operation ne sont donc pas visibles par les calculs contextuels pendant la validation.
- `SeaponyParadeEditorNoteTiming` a deja une logique de roll-chain: un roll suivi d'un roll finit sur le roll suivant, et le dernier roll ajoute un padding visuel pour revenir a un multiple de 4. Cette logique ne cree pas les hits manquants.

## Objectif fonctionnel
Permettre a une action d'editeur de creer plusieurs `ChartNote` reelles, donc plusieurs hits en gameplay, tout en traitant leur pose comme une seule intention d'editeur avec une fenetre editoriale globale/coherente.

Cas Seapony confirme:
- C'est toujours le dernier Roll de la serie qui autocomplete.
- Si la serie arrive a 5 rolls, le 5eme cree aussi les 3 hits manquants pour terminer a 8.
- Les notes ajoutees doivent rester de vraies notes de chart avec `action = seapony_parade_roll`.

## Plan d'implementation
1. Ajouter un contexte de placement editor
- Introduire un petit `EditorNotePlacementContext` contenant au minimum `double Crotchet` et `IReadOnlyList<ChartNote> ExistingNotes`.
- Changer `IEditorNotePlacementStrategy.CreatePlacements(...)` pour recevoir ce contexte au lieu du seul `crotchet`.
- Adapter `EditorNoteDefinition.CreatePlacements`, `SingleEditorNotePlacementStrategy` et `IntervalEditorNoteProvider.IntervalEditorNotePlacementStrategy`.
- But: les strategies speciales, comme Seapony, doivent pouvoir regarder les notes existantes avant/apres la note source.

2. Generaliser la generation multi-notes dans `BeatmapEditorElement`
- Remplacer l'usage direct de `BeatmapEditorElement.CreateIntervalPlacements()` par `_optionsDefinition.CreatePlacements(_optionsNote, new EditorNotePlacementContext(...))` pour les creations simples ET intervalle.
- Garder les cles `interval_duration_beats` / `interval_step_beats` sur la note source temporaire comme configuration de generation.
- Supprimer ou reduire le helper local `CreateIntervalPlacements()` et `CloneNoteForInterval()` pour eviter deux systemes concurrents.
- Adapter `DrawIntervalPreview()` pour utiliser exactement la meme strategie que la creation reelle.

3. Ajouter une strategie Seapony dediee
- Ajouter `.Placement(new SeaponyParadeEditorNotePlacementStrategy())` dans `SeaPonyParadeNoteEditor`.
- Pour `Swim` et `Tap Tap`, retourner une seule note clonee sans metadata d'intervalle.
- Pour `Roll`:
  - Generer les rolls demandes par la creation source: 1 hit en creation simple, ou N hits si la source contient une configuration d'intervalle.
  - Determiner la serie Roll autour de la source en regardant les rolls contigus dans l'ordre de chart: rolls avant, rolls generes, et rolls apres si on insere dans une serie existante.
  - Calculer `padding = (4 - totalRollCount % 4) % 4`.
  - Ajouter le padding apres le dernier roll de la serie combinee, avec un espacement de `stepBeats` si intervalle, sinon 1 beat.
  - Exemple: 4 rolls existants puis creation d'un Roll => total 5, creation du nouveau Roll + 3 rolls de padding.
  - Exemple: intervalle Roll de 5 hits sans serie existante => 5 hits demandes + 3 rolls de padding.
  - Nettoyer les cles de configuration editor-only avant de stocker les `ChartNote` dans la chart.

4. Corriger la validation pour la fenetre globale/contextuelle
- Dans `BeatmapEditorDocument.TryPlaceNotes()`, construire une liste temporaire triee `existing + placements` avant de tester les collisions.
- Faire calculer `GetBlockingStart/End` avec un `CreateRelativeNoteGetter` base sur cette liste temporaire, pas uniquement `Chart.Notes`.
- Adapter `FindBlockingNote` et `PlacementsBlockEachOther` pour utiliser ce contexte temporaire.
- Effet attendu: les rolls generes dans la meme operation se voient comme une chaine, donc leur hit window editoriale devient coherente/globalement contigue au lieu que chaque roll se croie seul/dernier.
- Ne pas changer le moteur rhythm/runtime: chaque `ChartNote` generee reste un hit normal avec sa propre reaction gameplay.

5. UI et feedback
- Garder le flux existant: placement simple avec `ENTER`, intervalle avec `I`.
- Le status doit annoncer le nombre reel de notes creees apres padding.
- Optionnel mais utile: dans le panel Seapony, afficher une ligne informative lorsque `Roll` est selectionne, indiquant que le dernier roll complete la serie au multiple de 4.

6. Verification
- Serie vide + Roll simple: cree 4 rolls au total.
- Serie de 4 rolls + Roll simple: cree le 5eme roll + 3 padding, soit 8 dans la serie.
- Intervalle Roll de 5 hits: cree 8 rolls.
- Intervalle Roll de 4 hits: cree 4 rolls, pas de padding.
- Insertion dans une serie existante: la serie finale doit rester multiple de 4 ou etre bloquee proprement si les notes de padding chevauchent autre chose.
- Swim/Tap Tap: pas de padding.
- Verifier les collisions editor et le rendu des hit windows sur la timeline.
- Build via sortie temporaire si les DLL debug sont verrouillees: `dotnet build --no-restore -o obj\\kilo-build`.
