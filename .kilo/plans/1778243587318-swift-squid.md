# Plan: facade SimpleRhythmGame<TAction>

## Objectif

Ajouter une couche SDK declarative au-dessus de `EditorNoteProvider` pour que l'ajout d'un rhythm game simple demande seulement:

- une enum d'actions;
- une classe `SimpleRhythmGame<TAction>`;
- une declaration de scene, note runtime et clips/patterns.

Le noyau actuel reste en place. `EditorNoteProvider`, `INotePayload`, `RuntimeNoteDraft`, `EditorNoteVariant`, les codecs et la migration legacy restent les internals bas niveau. `SeeSaw` reste sur l'API bas niveau. `Seapony Parade` devient le premier jeu porte sur la nouvelle facade.

## Constat codebase

- `EditorNoteProvider` fournit deja le bon point d'integration: `Definition`, `CreateClips`, `CompileClip`, `GetClipTypeIdFromLegacyNote`, `GetNoteVariantIndex`, `GetEditorStyle`, `CreateScene`.
- `EditorNoteDefinitions` decouvre automatiquement les providers concrets avec constructeur public sans parametre; une classe concrete derivee de `SimpleRhythmGame<TAction>` sera donc enregistree sans liste centrale.
- `EditorClipCompiler` delegue deja a `provider.CompileClip`, donc la facade peut compiler vers les `ChartNote` existantes sans changer le runtime.
- `RuntimeNoteDraft` sait deja convertir beat + hold vers `ChartNote` avec une `ChartTempoMap`.
- Le provider Seapony actuel contient exactement la plomberie a cacher: codec, payload, variants, clips, migration legacy et pattern compiler.
- `docs/creer-un-rhythm-game.md` documente aujourd'hui le chemin bas niveau; elle devra devenir une doc SDK simple avec une section avancee.

## Decisions techniques

- Pas de source generation: les objets sont generes au runtime par un builder generique.
- Les nouveaux fichiers vont dans `Elements/Editor/Simple/`, avec le namespace existant `MLP_RiM.Elements.Editor` pour garder `using MLP_RiM.Elements.Editor;`.
- Les identifiants par defaut sont derives de l'enum:
  - `Basic` -> `basic`;
  - `DoubleTap` -> `double_tap`;
  - action legacy -> `${gameId}_${action_id}`;
  - clip id -> `${gameId}.${action_id}`;
  - variant id -> `${action_id}`.
- Le codec generique accepte les payloads avec metadata explicite `_game`/`_type` uniquement si elle correspond, et accepte aussi les anciennes notes sans metadata si `action` correspond a une action connue.
- Les valeurs legacy custom restent possibles via une API d'alias, mais le chemin normal suit la convention ci-dessus.
- `SimpleRhythmGame<TAction>` construit sa definition en lazy-init, pas dans le constructeur, pour eviter d'appeler `Build` avant la fin de construction du provider derive.

## API cible

`SimpleRhythmGame<TAction>`:

```csharp
public abstract class SimpleRhythmGame<TAction> : EditorNoteProvider
    where TAction : struct, Enum
{
    protected abstract void Build(RhythmGameBuilder<TAction> game);
}
```

`RhythmGameBuilder<TAction>` expose:

- `.Id(string gameId)`
- `.DisplayName(string displayName)`
- `.SortOrder(int sortOrder)`
- `.Scene(Func<Scene> sceneFactory)`
- `.RuntimeNote(string noteId = "note")`
- `.Clip(TAction action)`
- `.NoHit(double defaultLengthBeats = 1)`
- `.LegacyActionValue(TAction action, string value)` pour les vieux formats non conventionnels

`RuntimeNoteBuilder<TAction>` expose:

- `.Input(string inputAction)`
- `.Hold(double beats)`
- `.Occupies(double beforeBeats, double afterBeats)`
- `.HitWindow(double beforeBeats, double afterBeats)`
- `.SameVariantHitWindow(double beforeBeats, double afterBeats)`
- `.Timing(IEditorNoteTiming timing)`

`SimpleClipBuilder<TAction>` expose:

- `.Id(string clipTypeId)`
- `.Name(string displayName)`
- `.Color(Color color)`
- `.Style(EditorVisualStyle style)`
- `.SingleHit()`
- `.Continuous(double defaultLengthBeats)`
- `.Field(EditorClipFieldDefinition field)` / `.Fields(...)`
- `.Data(string key, string value)` pour les rares donnees de clip supplementaires
- `.LeadIn(double beats)`
- `.Emit(double offsetBeats, double holdBeats = 0)`
- `.Pair(double offsetBeats)` comme raccourci pour deux emits `0` et `offsetBeats`
- `.RepeatEvery(double beats)`
- `.RepeatPairsEvery(double beats)` comme alias lisible de repetition du pattern d'emits courant
- `.PadToMultipleOf(int multiple)`
- `.HoldForClipLength()`
- `.Compile(Action<SimpleClipCompileContext<TAction>, SimpleRuntimeNoteEmitter<TAction>> compile)` comme escape hatch local

Regle importante: `.SingleHit()` et `.Continuous(...)` ne creent pas immediatement une emission explicite. Si aucun `.Emit(...)`/`.Pair(...)`/compiler custom n'est declare, le compiler ajoute implicitement une note a offset `0`. Cela evite le doublon dans `.SingleHit().Emit(0).Emit(0.5)`.

## Traduction vers l'existant

Au build final, `RhythmGameDefinition<TAction>` produit:

- `RhythmGameId` depuis `.Id(...)`;
- `RhythmGameDisplayName` depuis `.DisplayName(...)`;
- `NoteTypeId` depuis `(GameId, NoteId)`;
- un `EnumNoteCodec<TAction>`;
- un `EditorNoteDefinitionBuilder` configure avec input, timing, occupation, hit windows et `.Matches(note => codec.Matches(note?.AdditionnalData))`;
- un variant runtime par clip runtime declare;
- un `EditorClipDefinition` par clip declare;
- un clip `No Hit` seulement si `.NoHit(...)` est appele;
- le clip `Switch Game` reste fourni automatiquement par `EditorNoteProvider`.

`EnumNotePayload<TAction>` implemente `INotePayload` et ecrit:

```text
_game = game id
_type = note id
_version = 1
action = generated or aliased action value
```

`SimpleRhythmGame<TAction>` override/delegue:

- `SortOrder`
- `RhythmGameId`
- `RhythmGameDisplayName`
- `Definition`
- `CreateScene()`
- `CreateClips()`
- `CompileClip(...)`
- `GetClipTypeIdFromLegacyNote(...)`
- `GetNoteVariantIndex(...)`
- `GetEditorStyle(...)`

Les methodes avancees de `EditorNoteProvider` restent overrideables par un jeu simple si necessaire, mais ne sont pas dans le chemin normal.

## Compiler simple

`SimplePatternCompiler` compile un `SimpleClipDefinition<TAction>` en `RuntimeNoteDraft[]`:

- base beat = `clip.StartBeat + LeadInBeats`;
- longueur de pattern = `Math.Max(0, clip.LengthBeats)`;
- emits explicites = offsets declares;
- si aucun emit explicite, emit implicite a `0`;
- sans repetition, les emits sont produits une fois;
- avec `.RepeatEvery(step)`, le pattern d'emits est repete de `0` a `LengthBeats` inclus;
- si `LengthBeats <= epsilon`, tous les emits du pattern sont produits une fois, meme si un offset depasse `0`, pour permettre `DoubleTap` et `TapTap` single-hit;
- avec longueur positive, les emits dont `repeatOffset + emitOffset` depasse la longueur sont ignores;
- `.HoldForClipLength()` met `HoldBeats = clip.LengthBeats` sur les notes generees, sauf hold explicite sur un emit;
- `.PadToMultipleOf(n)` complete la serie apres le dernier hit avec le step de repetition.

Pour reproduire Seapony Roll, le padding doit regarder les notes existantes du meme action dans `NoteCompileContext.ExistingNotes` et calculer la serie contigue autour des notes generees, comme l'actuel `SeaponyParadePatternCompiler.GetCombinedRollSeries`.

## Placement note legacy/editor

Ajouter `SimpleEditorNotePlacementStrategy<TAction>` pour que la creation directe de notes via l'ancien flux options/intervalle continue de fonctionner:

- lit l'action dans `sourceNote.AdditionnalData` via le codec generique;
- utilise le beat source comme beat runtime, sans appliquer le `LeadIn` de clip auteur;
- utilise `PlacementOptions.RepeatDurationBeats` / `RepeatStepBeats` quand elles existent;
- convertit les `RuntimeNoteDraft` en clones de `sourceNote`, comme `SeaponyParadeEditorNotePlacementStrategy` le fait aujourd'hui.

Cela garde le comportement clip auteur propre tout en preservant les anciens chemins de creation directe.

## Portage Seapony

Remplacer `SeaponyParadeEditorNoteProvider : EditorNoteProvider` par `SeaponyParadeEditorNoteProvider : SimpleRhythmGame<SeaponyAction>`.

Conserver les constantes publiques utilisees par les tests et les charts:

- `GameId = "seapony_parade"`
- `SwimClipId = "seapony_parade.swim"`
- `RollClipId = "seapony_parade.roll"`
- `TapTapClipId = "seapony_parade.tap_tap"`
- `TypeId = new(GameId, "note")`

Declaration cible:

```csharp
protected override void Build(RhythmGameBuilder<SeaponyAction> game)
{
    game.Id(GameId)
        .DisplayName("Seapony Parade")
        .SortOrder(10)
        .Scene(() => new SeaPonyParade());

    game.RuntimeNote("note")
        .Input("ReactMain")
        .Occupies(1, 1)
        .HitWindow(0, 2)
        .Timing(new SeaponyParadeEditorNoteTiming());

    game.Clip(SeaponyAction.Swim)
        .Id(SwimClipId)
        .Name("Swim")
        .Color(Color.CornflowerBlue)
        .Continuous(2)
        .RepeatEvery(2);

    game.Clip(SeaponyAction.Roll)
        .Id(RollClipId)
        .Name("Roll")
        .Color(Color.DeepSkyBlue)
        .Continuous(3)
        .LeadIn(2)
        .RepeatEvery(1)
        .PadToMultipleOf(4);

    game.Clip(SeaponyAction.TapTap)
        .Id(TapTapClipId)
        .Name("Tap Tap")
        .Color(Color.LightBlue)
        .SingleHit()
        .LeadIn(2)
        .Pair(0.5)
        .RepeatPairsEvery(1.5);

    game.NoHit(1)
        .Color(Color.DimGray);
}
```

`SeaponyNotePayload` et `SeaponyNoteCodec` peuvent rester comme facade de compatibilite, deleguant au codec generique ou reutilisant la meme convention. Les tests existants et `SeaponyParadeEditorNoteTiming` peuvent ainsi rester stables.

Supprimer l'ancien `SeaponyParadePatternCompiler` seulement apres validation que plus aucune reference ne l'utilise. Sinon le laisser temporairement mais ne plus l'utiliser par le provider.

## Tests a ajouter ou adapter

Ajouter des tests unitaires autour de la nouvelle facade:

- codec generique: ecrit metadata + action, lit une note sans metadata, rejette metadata explicite d'un autre jeu;
- mapping ids: `DoubleTap` donne `double_tap`, action `my_game_double_tap`, clip `my_game.double_tap`;
- compile simple hit: une note au beat de depart;
- compile double emit: deux notes avec offsets `0` et `0.5`;
- compile lead-in + repeat: start decale et repetition inclusive;
- compile hold: `HoldForClipLength()` remplit `HoldBeats` et `HoldDuration` via `RuntimeNoteDraft`;
- legacy migration: une note runtime avec action sans metadata retourne le clip correspondant.

Conserver les tests Seapony existants comme regression:

- `LegacySeaponyRuntimeNoteMigratesToMatchingAuthorClip`;
- `ClipCompilerGeneratesContinuousTapTapAndNoHitRuntimeNotes`;
- `RollClipReservesTwoCueBeatsBeforeRuntimeActions`;
- `TapTapClipReservesTwoCueBeatsBeforeRuntimeActions`;
- undo/redo clips generes.

Commande de validation: `dotnet test MLP_RiM.Tests/MLP_RiM.Tests.csproj`.

## Criteres d'acceptation

- Un jeu simple peut tenir en moins de 80 lignes dans un seul fichier, hors scene runtime.
- Aucun codec ni payload manuel n'est requis pour un jeu enum-based standard.
- Les clips standards couvrent single hit, hold, double tap, repeat, lead-in et padding.
- Seapony Parade garde les memes beats generes qu'avant.
- SeeSaw n'est pas modifie.
- Tous les tests passent.
