# Plan: correction progression BG Seapony

Le compteur dans la factory du `VisualNoteManager` ne doit pas etre utilise : en cas de seek/rewind, le manager peut recreer des visual notes et donc reincrementer le compteur, ce qui explique les destinations abusivement hautes.

1. Modifier `Gameplay/Visual Notes/SeaponyBgVisualNote.cs` pour que la destination ne soit plus calculee depuis `_background.Progression` dans le constructeur.
2. Ajouter un parametre `int backgroundScrollDestinationBeat` au constructeur et l'assigner directement a `_backgroundScrollDestinationBeat`.
3. Supprimer le `Console.WriteLine` de debug dans `handleSwim` pour eviter le spam et ne pas garder de trace temporaire.
4. Dans `Scenes/SeaPonyParade.cs`, ajouter un helper prive `getBackgroundScrollDestinationBeat(Note note)` qui calcule l'index stable de la note dans `GLOBALS.beatmapPlayer.ChartPlayer.Notes`.
5. Le helper parcourt les notes triees par ordre de beatmap, compte uniquement les notes dont `AdditionnalData["action"] == "seapony_parade_swim"`, et retourne le compteur quand il atteint exactement l'instance `note` recue par la factory. Ainsi la 1ere note swim vaut `1`, la 2eme `2`, la xeme `x`, meme apres seek/rewind.
6. Utiliser ce helper dans la creation de `_infiniteScrollBgVisualNotes` : `new SeaponyBgVisualNote(note, ..., _infiniteScrollBg, getBackgroundScrollDestinationBeat(note), ...)`.
7. Ajouter `using Rhythm.Note;` dans `SeaPonyParade.cs` si necessaire pour typer le helper.
8. Verifier qu'aucun autre appel au constructeur `SeaponyBgVisualNote` n'existe et lancer une compilation rapide si possible.
