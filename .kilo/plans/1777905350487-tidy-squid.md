# Plan: Seapony Parade Tap Tap

## Objectif
Ajouter le pattern `seapony_parade_tap_tap` pour l'editeur et le runtime Seapony Parade, avec cues de depart/fin, paires de taps, pause entre paires, alternance `pretap`/`posttap`, et miroir des poneys 2 et 4.

## Constat actuel
- `Tap Tap` est deja declare dans `SeaPonyParadeNoteEditor` et `SeaponyParadeNoteOptionsPanel` avec `action = seapony_parade_tap_tap`.
- `SeaPonyParade.cs` ne reconnait actuellement que `seapony_parade_swim` et `seapony_parade_roll`.
- `SeaponyVisualNote.cs` ne gere que `swim` et `roll`.
- Les states `pretap` et `posttap` existent deja dans la state machine avec `Template_pony_tap1` et `Template_pony_tap2`.
- Le placement/timing specifique Tap Tap n'est pas encore implemente.

## Clarification verrouillee
- Les poneys miroirs sont les 2 et 4, donc indices zero-based `1` et `3` dans le code.
- Le miroir est a appliquer comme un miroir sur axe vertical, donc un flip horizontal visuel. Si la lib n'expose pas de `SpriteEffects`, utiliser `Scale.X = -SCALE` et restaurer `Scale.X = SCALE` hors Tap Tap.

## Plan d'implementation
1. Ajouter `TapTapAction = "seapony_parade_tap_tap"` dans les classes qui dispatchent les actions Seapony.
2. Etendre `TryGetSeaponyAction`, `tryGetAction`, `createVisualNotes`, `handleNoteReacted`, `getSeaponyApproachDuration` et `getSeaponyDespawnDelay` pour accepter Tap Tap.
3. Cote editeur, modifier `SeaponyParadeEditorNotePlacementStrategy`:
   - Garder le comportement simple si aucune configuration intervalle n'est fournie: une note Tap Tap a la position source.
   - Si une configuration intervalle est fournie, generer les hits en paires: premier hit a `start`, second a `start + 0.5 beat`, puis pause de `1 beat` avant la prochaine paire.
   - Continuer les paires jusqu'a la duree intervalle, sans generer de notes pour les cues car les cues seront implicites au runtime/visual.
   - Preserver `AdditionnalData["action"] = TapTapAction` sur toutes les notes generees et supprimer les donnees intervalle comme les autres placements.
4. Cote timing editeur dans `SeaponyParadeEditorNoteTiming`:
   - Premiere note d'une serie Tap Tap: start a `SongPosition - 2 beats` pour couvrir `Cue Cue` avant les hits.
   - Notes Tap Tap consecutives: start a `SongPosition`, end a la note Tap Tap suivante.
   - Derniere note Tap Tap: end a `SongPosition + 2 beats` pour couvrir le `Cue Cue` final.
   - Hit window Tap Tap: start a `SongPosition`, end a la note Tap Tap suivante si elle existe, sinon `SongPosition + 0.5 beat` ou le cue final selon l'affichage le plus lisible.
   - Same-variant window: `SongPosition` a `SongPosition + 0.5 beat`, pour autoriser les hits espaces de 0.5 beat.
5. Cote `SeaPonyParade.cs`, ajouter des helpers Tap Tap similaires aux helpers roll:
   - `getTapTapIndexInSequence(note)` pour compter les Tap Tap consecutifs avant la note.
   - `getTapTapHitsRemainingInSequence(note)` ou equivalent pour detecter la derniere note.
   - `getTapTapDespawnDelay(note)` pour aller jusqu'a la prochaine note Tap Tap, ou `2 * crotchet` sur la derniere note.
   - Optionnellement factoriser les tests d'action pour eviter de dupliquer les checks `AdditionnalData`.
6. Cote `SeaponyVisualNote.cs`, ajouter les donnees constructeur necessaires pour Tap Tap:
   - Index dans la sequence.
   - Nombre restant ou booleen derniere note.
   - Provider de reaction reussie pour le poney joueur, comme `swim`.
7. Ajouter `handleTapTap(...)` dans `SeaponyVisualNote.cs`:
   - Pendant le premier cue de la premiere note, orienter les poneys avec leur binome: poneys 1 et 3 inchanges, poneys 2 et 4 en miroir.
   - Avant le premier hit, poser les states initiaux: poneys 1 et 3 en `pretap`, poneys 2 et 4 en `posttap`.
   - A chaque hit, inverser `pretap`/`posttap` selon la parite du nombre de hits passes.
   - Pour le poney joueur, appliquer l'inversion post-hit seulement si la note a ete reussie, afin de garder une logique proche de `swim`/`roll`.
   - Pour les autres poneys, appliquer l'animation automatiquement.
   - Jouer les SFX de cue et de hit uniquement via `_seaPonyIndex == 0`, comme les handlers existants.
8. Proteger les resets visuels:
   - Restaurer `Scale = Vector2.One * SCALE` ou l'equivalent positif quand un poney redevient idle ou sort du Tap Tap.
   - Restaurer rotation et scale dans `resetSeaPonyTimelineState`.
   - S'assurer que `applyBaseStateForIdleSeaPonies` ne laisse pas un miroir Tap Tap actif.
9. Verification apres implementation:
   - Lancer `dotnet build`.
   - Tester manuellement une sequence: `Cue Cue Hit Hit Pause Hit Hit Cue Cue`.
   - Tester un rewind au milieu de la sequence.
   - Tester dans l'editeur le placement d'une seule note Tap Tap et d'une serie intervalle.

## Points d'attention
- Les cues restent implicites pour eviter d'ajouter un type de note non jouable.
- La logique de sequence doit se baser sur les notes Tap Tap consecutives dans le chart, comme le roll, afin de ne pas casser les pauses de 1 beat.
- Si la lib graphique expose deja un flag de flip horizontal, preferer ce flag a `Scale.X` negatif.
