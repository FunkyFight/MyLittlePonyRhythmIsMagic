# Plan: interval editor note

## Contexte
- L'editeur expose des types via `EditorNoteDefinition`, `IEditorNoteProvider` et `EditorNoteDefinitions`.
- `BeatmapEditorDocument.TryPlaceNote` ajoute actuellement une seule `ChartNote` apres validation d'occupation.
- Les options de creation passent par une `ChartNote` temporaire (`_optionsNote`) et son `AdditionnalData`.
- Le format de beatmap ne contient que des `ChartNote`; l'interval note doit donc etre une note d'editeur/macro qui expand en plusieurs `ChartNote`, pas une entite sauvegardee distincte.

## Objectif
Ajouter un nouveau type d'editor note: `Interval`.
- Debut: position snappee courante lors de la creation.
- Fin: configuree comme une duree en beats/crotchets depuis le debut.
- Intervalle: configure en crotchets.
- Generation: un hit a `start`, puis tous les `intervalBeats * crotchet` jusqu'a `end` inclus, avec tolerance flottante.
- Chaque hit genere est une vraie `ChartNote` sauvegardee.
- L'implementation doit etre solide et heritable pour permettre des interval notes specialisees plus tard.

## Design propose
1. Introduire une abstraction de placement multiple.
   - Ajouter un contrat type `IEditorNotePlacementStrategy` ou equivalent simple sur `EditorNoteDefinition`.
   - Strategie par defaut: cree et place une seule `ChartNote`, comportement actuel.
   - Strategie interval: cree une sequence de `ChartNote` depuis une note de configuration.
   - Garder l'API existante autant que possible, mais ajouter une methode document qui place une liste atomiquement.

2. Rendre le placement atomique dans `BeatmapEditorDocument`.
   - Ajouter une methode interne/publique type `TryPlaceNotes(EditorNoteDefinition definition, IReadOnlyList<ChartNote> notes, out IReadOnlyList<ChartNote> placedNotes, out string reason)`.
   - Valider toutes les notes avant mutation.
   - Refuser si une note generee bloque une note existante.
   - Refuser si deux notes generees se bloquent entre elles.
   - N'ajouter les notes, trier et marquer dirty qu'apres validation complete.
   - Conserver `TryPlaceNote` comme wrapper single-note pour les usages actuels.

3. Ajouter une base heritable pour les interval notes.
   - Nouveau dossier probable: `Elements/Editor/Notes/Interval/`.
   - Creer une classe abstraite type `IntervalEditorNoteProvider : EditorNoteProvider`.
   - La base expose des constantes/cles robustes dans `AdditionnalData`, par exemple:
     - `interval_duration_beats`
     - `interval_step_beats`
   - La base fournit parsing/clamping centralise:
     - duree minimale `0` ou `step`, selon decision d'implementation;
     - intervalle strictement positif;
     - valeurs invalides fallback sur des defaults.
   - La base fournit la generation inclusive des temps:
     - `time = start + i * stepSeconds`
     - stop a `end + epsilon`
     - clamp a `>= 0`
   - La base expose une methode virtuelle type `CreateHitNote(double songPosition, double crotchet, ChartNote sourceConfigNote)` pour specialiser les hits generes.

4. Ajouter le provider concret `RhythmInputIntervalEditorNote`.
   - Ajouter `EditorNoteKind.Interval` dans l'enum.
   - Definition: display name `Interval`, input action `ReactMain`.
   - `CreateHitNote` genere des notes rythmiques simples:
     - `SongPosition = generatedTime`
     - `HoldDuration = 0`
     - `InputActionToPress = ReactMain`
     - `AdditionnalData` vide ou une petite marque editor non bloquante si necessaire.
   - `Matches` ne doit pas reconnaitre les hits generes comme `Interval`, car l'interval note n'est pas persistante; les hits restent des chart notes normales.

5. Ajouter un panel d'options interval.
   - Creer `IntervalEditorNoteOptionsPanel`.
   - Options proposees avec l'UI actuelle:
     - `DURATION` en beats/crotchets via dropdown: par exemple `1, 2, 4, 8, 16, 32`.
     - `INTERVAL` en beats/crotchets via dropdown: par exemple `0.25, 0.5, 1, 2, 4`.
     - un title informatif montrant le nombre de hits generes.
   - Stocker les valeurs dans `_optionsNote.AdditionnalData` pendant la creation.
   - Si un choix plus libre devient necessaire, etendre ensuite `DevUiWindowRow` avec un numeric stepper/text field; pour cette implementation, les dropdowns restent minimaux et compatibles avec l'UI existante.

6. Brancher le nouveau type dans l'editeur.
   - Ajouter le provider a `EditorNoteDefinitions.Providers`.
   - Ajuster `BeatmapEditorElement.CreatePendingNote`:
     - demander a la definition/strategie les notes a placer;
     - appeler la methode atomique du document;
     - mettre a jour `_lastCreatedNoteData` avec les options interval;
     - afficher un statut du type `Created Interval: N hits from ... to ...`;
     - ouvrir les options sur la premiere note generee seulement si elle a un panel pertinent; sinon fermer ou garder un statut clair.
   - Ajouter une couleur dediee dans `GetNoteColor` pour `Interval`/hits preview si necessaire. Comme les hits generes ne matchent pas `Interval`, la couleur de timeline restera celle des chart notes simples sauf si on marque les hits.

7. Robustesse details
   - Utiliser `CultureInfo.InvariantCulture` pour parser/serialiser les doubles dans `AdditionnalData`.
   - Eviter les boucles infinies si intervalle invalide ou crotchet <= 0.
   - Utiliser un epsilon pour l'inclusion de fin (`1e-6` ou proche de `HitWindowEpsilonSeconds`).
   - Les notes generees doivent etre des instances distinctes avec dictionnaires distincts.
   - Ne pas supprimer ou modifier les notes existantes en cas d'echec partiel.

8. Verification
   - `dotnet build`.
   - Test manuel attendu dans l'editeur:
     - selectionner `Interval` avec UP/DOWN;
     - ENTER a une position snappee;
     - configurer duration/interval;
     - CREATE;
     - verifier que le compteur NOTES augmente du nombre de hits attendu;
     - sauvegarder/recharger et confirmer que seules les `ChartNote` generees persistent.

## Decisions retenues
- L'interval note est une macro d'edition, pas un nouveau format persiste.
- Les hits generes par le provider concret sont des notes rythmiques simples `ReactMain`.
- L'architecture reste heritable via une classe abstraite interval et une methode virtuelle de creation de hit, pour creer plus tard des intervals qui generent des SeeSaw notes ou d'autres types.
