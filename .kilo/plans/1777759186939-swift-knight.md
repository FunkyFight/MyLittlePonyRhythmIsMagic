# Plan: Improve Beatmap Editor Loading, New Beatmap Modal, and BeatmapPlayer State

## Goals
- Fix `CTRL+L` in `Elements/Editor/BeatmapEditorElement.cs` so it actually reloads the currently selected/open chart instead of appearing to do nothing.
- Add `CTRL+N` to create a blank beatmap through a centered modal prompt asking for the beatmap name, with a create button, using/expanding `Elements/DevUI/`.
- Add a boolean chart-loaded state to `Gameplay/BeatmapPlayer.cs` that is true when a chart is loaded.

## Implementation Steps

1. Update `Gameplay/BeatmapPlayer.cs`
- Add a public read-only property named `HasAChartLoaded` (or `hasAChartLoaded` only if matching the exact requested casing is preferred, but C# style suggests `HasAChartLoaded`).
- Set it to `true` whenever a chart is assigned in:
  - `StartMetronomeDebugMap`
  - `StartSeeSawDebugMap`
  - `StartBeatmap`
  - `StartBeatmapPaused`
- Set it to `false` in `DisposeConductor()` when `CurrentChart`/playback state is cleared.
- Also clear `CurrentChart = null` in `DisposeConductor()` so the property remains consistent with actual loaded state.

2. Fix editor load behavior
- Inspect current behavior: `Load()` always calls `BeatmapEditorDocument.LoadOrCreate(_defaultSongPath, _defaultChartPath, 100)`, so `CTRL+L` reloads the default paths instead of the selected/current chart path.
- Refactor loading so there is a method like `LoadDocument(string songPath, string chartPath)`.
- On startup, load the defaults as before.
- On `CTRL+L`, reload using `_document.ChartPath` and `_document.SongPath` when `_document` exists; otherwise fallback to defaults.
- Refresh song/chart lists after loading and resync selected indexes.
- Rebuild playback and status after reload.

3. Add modal support in `Elements/DevUI/`
- Extend `DevUiFloatingWindow` minimally to support modal-like centered windows with text rows/input display and buttons, or add enough generic row support to avoid editor-specific drawing code.
- Prefer minimal additions:
  - Add a `TextInput`/`Prompt` row kind if needed, or reuse `Title` plus a button row and render the typed buffer in `BeatmapEditorElement` via rows.
  - If generic text input support would become too broad, keep input handling in `BeatmapEditorElement` and use DevUI rows for display/button.
- Ensure clicking outside the new beatmap modal does not accidentally create/cancel unless explicitly intended. Escape should cancel.

4. Add new beatmap flow in `BeatmapEditorElement.cs`
- Add fields:
  - `DevUiFloatingWindow _newBeatmapWindow`
  - `bool _isCreatingNewBeatmap`
  - `string _newBeatmapNameBuffer`
- Initialize `_newBeatmapWindow` in the constructor.
- In command handling:
  - Detect `CTRL+N` before normal text edit handling if appropriate.
  - Open modal centered on screen, prefilled with something like `New Beatmap` or empty.
  - While modal is open, route keyboard input to the new beatmap name buffer and block normal editor commands.
  - `Enter` or create button triggers creation; `Escape` cancels; `Backspace` edits; reuse/extend `TryKeyToChar` for text input.
- Creation behavior:
  - Sanitize the entered beatmap name into a safe filename for `Beatmaps/<name>.xml`.
  - Create a new `BeatmapEditorDocument` via `CreateNew(_document?.SongPath ?? _defaultSongPath, newChartPath, bpm)`.
  - Set metadata beatmap name from the prompt.
  - Reset `_manualSongPosition` to 0.
  - Refresh charts, sync selected index, rebuild playback, set dirty status.
  - Keep it unsaved until `CTRL+S`, unless immediate file creation is preferred. Given “créer une nouvelle beatmap vierge”, use an in-memory blank document and let `CTRL+S` persist it, while `ChartPath` is already set.

5. Draw/update modal
- In `Update`, if `_isCreatingNewBeatmap`, update keyboard input and `_newBeatmapWindow.Update(...)` rows, then return before editor commands.
- In `Draw`, draw the editor as usual and then draw the centered modal above it.
- Modal rows should include:
  - Title/instruction: `New beatmap name`
  - Current text buffer line
  - `CREATE` button
- Use a centered rectangle based on viewport dimensions, e.g. `420x150`.

6. Validation
- Build the project with the existing build command (`dotnet build` if applicable after checking project files).
- Manually reason-check keyboard conflicts:
  - `CTRL+N` opens modal.
  - `CTRL+L` reloads current chart.
  - Text input modal blocks normal editor shortcuts.
  - Existing metadata edit mode still works.
- Verify no unrelated files are modified.

## Notes / Decisions
- Use `HasAChartLoaded` as idiomatic C# property casing unless exact lower-case field/property is required.
- New beatmap should initially use the currently selected song path and current BPM when available.
- Blank beatmap should not overwrite an existing chart silently; if `Beatmaps/<name>.xml` exists, append a numeric suffix like `name_2.xml` or report status. Prefer appending suffix for smoother UX.
